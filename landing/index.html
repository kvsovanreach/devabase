Core Features

Everything you need, nothing you don't

One self-hosted backend replaces your vector database, auth service, file storage, chat API, and real-time infrastructure. All tightly integrated.

Vector Database

Store and query millions of high-dimensional vectors with sub-millisecond latency. Built on pgvector with automatic HNSW indexing and optimization.

  • Cosine, L2, and Inner Product distance metrics
  • Automatic HNSW index creation & tuning
  • Metadata filtering on vector queries
  • Upsert, search, and delete operations

Document Processing

Upload PDFs, Word docs, text files, and Markdown. Automatic chunking, embedding generation, and knowledge extraction — all in the background.

  • PDF, DOCX, TXT, Markdown support
  • Smart chunking with split & merge control
  • Background async processing pipeline
  • File storage with upload/download

Semantic & Hybrid Search

Vector search, BM25 keyword search, and hybrid retrieval with Reciprocal Rank Fusion and cross-encoder reranking.

  • Hybrid search (vector + BM25 with RRF)
  • Cross-encoder reranking for precision
  • Multi-collection search in one query
  • Configurable vector/keyword weight blending

RAG Chat & Streaming

Production-ready Retrieval-Augmented Generation with multi-turn conversation memory, source attribution, and real-time streaming via SSE.

  • Multi-turn conversations with history
  • Source attribution per response
  • Server-Sent Events streaming
  • Single or multi-collection RAG

App User Authentication

Complete auth system for your application's end-users. Registration, login, password reset, email verification, and dual-auth for row-level security.

  • JWT access & refresh tokens
  • Password reset & account lockout
  • Dual-auth: API key + user identity (RLS)
  • Token introspection & user management

Tables & REST API

Create PostgreSQL tables with auto-generated REST endpoints. Full CRUD, filtering, sorting, pagination — like a headless database.

  • Auto-generated REST for every table
  • Rich filtering, sorting, and pagination
  • Column definitions with type safety
  • Direct SQL query execution

Knowledge Graphs

Automatically extract entities and relationships from your documents. Build connected knowledge for graph-enhanced RAG retrieval.

  • Automatic entity extraction (NER)
  • Relationship mapping between entities
  • Graph traversal with configurable depth
  • Graph-enhanced retrieval for better RAG

Advanced Retrieval Strategies

State-of-the-art retrieval strategies that dramatically improve RAG accuracy beyond basic vector search.

  • HyDE (Hypothetical Document Embeddings)
  • Multi-Query expansion
  • Self-Query with contextual compression
  • Parent-Child chunking
And More

A complete platform, not just a database

Real-Time Events

WebSocket-based real-time event streaming. Subscribe to document, collection, table, and vector changes as they happen.

  • WebSocket with channel subscriptions
  • Document, collection, table, vector events
  • Project-scoped event filtering

Provider Management

Configure LLM, embedding, and reranking providers per project. Swap between OpenAI, Anthropic, Cohere, Voyage AI, and more.

  • LLM, embedding, and rerank providers
  • Per-project provider configuration

Webhooks

  • Delivery logs & test endpoints

RAG Evaluation & Benchmarks

Measure and improve your RAG pipeline quality. Run academic benchmarks (BEIR datasets), compare configurations, and export results.

  • BEIR & custom dataset benchmarks
  • Configuration comparison
  • Exportable evaluation results

File Storage

Upload, download, and manage files associated with your project. Integrated with document processing for seamless ingestion.

  • File upload & download API
  • Linked to document processing
  • Project-scoped file management

Prompt Templates & Admin

Manage prompt templates, monitor cache performance, track API usage analytics, and administer your platform from a single interface.

  • Reusable prompt template management
  • Cache stats & invalidation
  • Usage analytics & export
How It Works

From zero to production in minutes

Self-host Devabase with Docker and start building with the TypeScript SDK.

01

Deploy Devabase

Run the server with Docker or build from source. PostgreSQL with pgvector included.

docker compose up -d
02

Install the SDK

Add the fully-typed TypeScript SDK to your project.

npm install devabase-sdk
03

Configure & Ingest

Create a project, set up providers, create collections, and upload documents.

await db.documents.upload('docs', { file, filename })
04

Search, Chat & Build

Query your knowledge base with semantic search, RAG chat, or direct SQL.

await db.chat.stream({ collection: 'docs', message: '...' }, callbacks)

// Semantic search with reranking
const results = await db.search.query({
  collection: 'knowledge-base',
  query: 'authentication best practices',
  top_k: 10,
  rerank: true,
  filter: { category: 'security' }
});

// Hybrid search (vector + BM25)
const hybrid = await db.search.hybrid(
  'knowledge-base',
  { query: 'OAuth implementation', top_k: 10 }
);

// Multi-collection search
const multi = await db.search.multi({
  collections: ['docs', 'faq', 'tutorials'],
  query: 'How do I get started?'
});
// RAG chat with streaming
await db.chat.stream({
  collection: 'docs',
  message: 'How do I implement OAuth?'
}, {
  onSources: (s) => console.log(s),
  onContent: (chunk) => process.stdout.write(chunk),
  onDone: (id, tokens) => console.log('Done!')
});

// Multi-turn conversations
const r1 = await db.chat.send({
  collection: 'docs', message: 'What is auth?'
});
const r2 = await db.chat.continue(
  r1.conversation_id, 'How about refresh tokens?'
);
// Create a table with typed columns
await db.tables.create({
  name: 'articles',
  columns: [
    { name: 'title', type: 'TEXT' },
    { name: 'views', type: 'INTEGER', default: 0 }
  ]
});

// CRUD with filtering & pagination
const rows = db.tables.rows('articles');
await rows.insert({ title: 'Hello', views: 0 });
const page = await rows.query({
  filter: { views: { gt: 100 } },
  sort: 'views:desc', limit: 20
});

// Or run raw SQL
const data = await db.sql.execute('SELECT * FROM articles');
// Register app users
const auth = await db.appAuth.register({
  email: 'user@example.com',
  password: 'securePassword123',
  name: 'John Doe'
});

// Login & get JWT tokens
const tokens = await db.appAuth.login({
  email: 'user@example.com',
  password: 'securePassword123'
});

// Set user context for RLS
db.asUser(tokens.access_token);
// Queries now respect row-level security
const myArticles = await db.tables
  .rows('articles').query();
// Connect to real-time events
db.realtime.connect({
  onOpen: () => {
    db.realtime.subscribe([
      'document:*',
      'collection:knowledge-base',
      'table:articles'
    ]);
  },
  onEvent: (event) => {
    console.log(event.channel, event.event);
    console.log(event.data);
  },
  onError: (msg) => console.error(msg)
});
Architecture

Built for performance, scale, and reliability

Written in Rust with Axum and PostgreSQL. Designed to run anywhere — from a single VPS to a distributed cluster.

Rust + Axum

Memory-safe, zero-cost abstractions, and async I/O for sub-50ms response times at scale.

PostgreSQL + pgvector

Your vectors, documents, tables, and auth all live in one battle-tested database.

Secure by Default

JWT auth, API key scoping, row-level security, Argon2 hashing, and project isolation.

Multi-Tenant

Isolated projects with separate data, API keys, providers, and configurations.

Self-Hosted & Open Source

Deploy on your own infrastructure. No vendor lock-in. Full control over your data.

Pluggable Providers

Swap LLM, embedding, and reranking providers without changing application code.
